b:T1337,<h2 id="installation"><a aria-hidden="true" tabindex="-1" href="#installation"><a href="#installation" style="margin-right: 10px">#</a></a>Installation</h2>
<p>First install the package eslint:</p>
<pre><code>yarn add eslint --only=dev
</code></pre>
<p>Then initialize init and it will walk you through a typical setup:</p>
<pre><code>yarn eslint --init
</code></pre>
<p>These are the options I selected. I like to start with the AirBnB template:</p>
<pre><code>? How would you like to configure ESLint? Use a popular style guide
? Which style guide do you want to follow? Airbnb
? Do you use React? Yes
? What format do you want your config file to be in? JSON
</code></pre>
<p>The airbnb template is already set up to use react and it will install all sorts of nice dependencies for you.</p>
<p>And thats basically it!</p>
<h2 id="configuration"><a aria-hidden="true" tabindex="-1" href="#configuration"><a href="#configuration" style="margin-right: 10px">#</a></a>Configuration</h2>
<p>It creates a file called <code>eslintrc.json</code> (the file will be different depending on which format you chose). And in this file will be your configuration.</p>
<p>If you open that file it will look like this:</p>
<pre><code>{
  "extends": "airbnb"
}
</code></pre>
<p>So it is basically just copying all of airbnb's rules. So if we want to now change the rules we are using we need to manually add them. We can turn off or change any rule and any rule we specify in our file will override the airbnb rule.</p>
<p>Here is an example of how to disable a rule called <code>no-underscore-dangle</code> and change the rule called <code>react/jsx-filename-extension</code> which requires all javascript files what use jsx to have the extension <code>.jsx</code>. This rule change also allows you to have <code>.js</code> extension.</p>
<pre><code>{
  "extends": "airbnb",
  "rules": {
    "react/jsx-filename-extension": [1, { "extensions": [".js", ".jsx"] }],
    "no-underscore-dangle": 0
  }
}
</code></pre>
<p>To disable eslint totally on a specific line, you can add this at the end of the line:</p>
<pre><code>var foo = "bar" // eslint-disable-line
</code></pre>
<p>And to disable 1 rule on that line you can add the rule like this:</p>
<pre><code>var foo = "bar" // eslint-disable-line no-alert
</code></pre>
<p>You can also do it this way:</p>
<pre><code>// eslint-disable-next-line
var foo = "bar"

// eslint-disable-next-line no-alert
var foo = "bar"
</code></pre>
<p>You can lint any file by running <code>yarn eslint path/to/file.js</code>.</p>
<p>You can also configure your editor to display any lint issues inline for you.</p>
<p>You can also set up a git pre-commit script to run before each commit which will enforce the lint rules in a very strong way.</p>
<h2 id="setting-up-a-eslint-pre-commit-git-hook"><a aria-hidden="true" tabindex="-1" href="#setting-up-a-eslint-pre-commit-git-hook"><a href="#setting-up-a-eslint-pre-commit-git-hook" style="margin-right: 10px">#</a></a>Setting up a ESLint pre-commit git hook</h2>
<p>I'm assuming you already have a git repo setup.</p>
<p>Create a new file:</p>
<pre><code>touch .git/hooks/pre-commit
</code></pre>
<p>Make the file executable:</p>
<pre><code>chmod +x .git/hooks/pre-commit
</code></pre>
<p>Then paste in this code here into that file:</p>
<pre><code>#!/bin/sh

STAGED_FILES=$(git diff --cached --name-only --diff-filter=ACM | grep ".jsx\{0,1\}$")
ESLINT="$(git rev-parse --show-toplevel)/node_modules/.bin/eslint"

if [[ "$STAGED_FILES" = "" ]]; then
  exit 0
fi

PASS=true

printf "\nValidating Javascript:\n"

# Check for eslint
if [[ ! -x "$ESLINT" ]]; then
  printf "\t\033[41mPlease install ESlint\033[0m (npm i --save-dev eslint)"
  exit 1
fi

for FILE in $STAGED_FILES
do
  "$ESLINT" "$FILE"

  if [[ "$?" == 0 ]]; then
    printf "\t\033[32mESLint Passed: $FILE\033[0m"
  else
    printf "\t\033[41mESLint Failed: $FILE\033[0m"
    PASS=false
  fi
done

printf "\nJavascript validation completed!\n"

if ! $PASS; then
  printf "\033[41mCOMMIT FAILED:\033[0m Your commit contains files that should pass ESLint but do not. Please fix the ESLint errors and try again.\n"
  exit 1
else
  printf "\033[42mCOMMIT SUCCEEDED\033[0m\n"
fi

exit $?
</code></pre>
